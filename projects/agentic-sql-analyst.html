Project 04 / Agentic analytics

Agentic SQL Analyst

A self-directed analytics agent that writes its own SQL to compare month-over-month performance and explain what changed — and why.

Working app
Python Streamlit Snowflake Azure AI Foundry

01 Problem

The recurring "how did we do vs. last month" question normally meant an analyst manually writing comparison SQL and rebuilding the same charts every single cycle — a repetitive, error-prone task that consumed hours of skilled analyst time on work that followed the same pattern every month.

02 Solution

An LLM agent reads a versioned "skills" playbook — data model, SQL patterns, and analysis templates — at runtime, dynamically generates SQL, executes it against Snowflake, and produces KPI deltas, waterfall charts, and narrative insights, with zero hardcoded SQL or business logic in the application itself. The agent also supports natural-language follow-up questions on top of the initial comparison.

03 Architecture

A chat UI passes the user's question to an Agent Orchestrator, which reads the versioned skills/ playbook, calls an LLM (Snowflake Cortex or Azure, swappable) to generate SQL, executes that SQL on Snowflake, and renders the result as KPIs, charts, and narrative — closing the loop back to the chat for follow-up questions.

Fig. 1 — Orchestrator reads the skills playbook, delegates SQL generation to a swappable LLM, executes on Snowflake, renders results back to chat.

04 Tech stack

Python Streamlit Snowflake Swappable LLM (Cortex / Azure AI Foundry) Docs-as-knowledge-base "skills" architecture

05 Highlights

  • All domain knowledge lives in editable markdown playbooks — no hardcoded SQL anywhere in the app.
  • Conversational follow-ups, e.g. "Why did GP drop for Customer X?"
  • Side-by-side comparison views plus waterfall charts showing the change itself.
