Project 02 / Personal platform engineering

Agentic App Workspace

A personal starter framework that distills the architecture and "context engineering" patterns from four production AI apps into a one-command scaffold for the next one.

Active use
Python Snowflake Templated archetypes AI-agent instructions

01 Problem

Every new Snowflake-backed AI app was re-deriving the same architecture decisions from scratch — config patterns, auth, agent orchestration, and documentation for AI coding assistants. That repeated work meant new projects spent their first weeks rebuilding infrastructure instead of solving the actual problem.

02 Solution

A workspace that distills patterns mined from four production apps into reusable archetypes — Streamlit-in-Snowflake, container agent app, full-stack React, research platform, and table chatbot — each fully self-contained with its own docs, config conventions, and AI-agent instructions, spun up through a single scaffold script. This is framed as personal platform engineering rather than a single user-facing app: it's a framework for building the other apps faster.

03 Architecture

A single scaffold script takes an archetype name and a target directory, then copies that self-contained archetype — its docs, config conventions, and AI-agent instructions — into a new, ready-to-build app repository.

Fig. 1 — One scaffold command selects an archetype and produces a new, fully self-contained app repository.

Every layer of the workspace follows the same three-part "harness" pattern I use for structuring any AI-agent workspace: a map (CLAUDE.md / copilot-instructions.md) that's always loaded and kept under 200 lines, a router (CONTEXT.md) that tells an agent what to load for a given task and what to skip, and the workspace itself — self-contained and siloed, so an agent working inside one archetype never reads another archetype's docs.
