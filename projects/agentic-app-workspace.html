Project 06 / Internal platform engineering

Agentic App Workspace

An internal starter framework that distills the architecture and "context engineering" patterns from four production AI apps into a one-command scaffold for the next one.

Active internal foundation · private
Python Snowflake Templated archetypes AI-agent instructions

01 Problem

Every new Snowflake-backed AI app was re-deriving the same architecture decisions from scratch — config patterns, auth, agent orchestration, and documentation for AI coding assistants. That repeated work meant new projects spent their first weeks rebuilding infrastructure instead of solving the actual problem.

02 Solution

A workspace that distills patterns mined from four production apps into reusable archetypes — Streamlit-in-Snowflake, container agent app, full-stack React, research platform, and table chatbot — each fully self-contained with its own docs, config conventions, and AI-agent instructions, spun up through a single scaffold script. This is framed as internal platform engineering rather than a single user-facing app: it's a framework for building the other apps faster.

03 Architecture

A single scaffold script takes an archetype name and a target directory, then copies that self-contained archetype — its docs, config conventions, and AI-agent instructions — into a new, ready-to-build app repository.

Fig. 1 — One scaffold command selects an archetype and produces a new, fully self-contained app repository.
